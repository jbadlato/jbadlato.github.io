OChem Buddy
Previous Next

What: A study aid for organic chemistry students that names simple organic molecules.

Why: To learn JavaScript, HTML, CSS, and Heroku.

How: I built this web app from scratch, using the HTML canvas element to allow the user to draw organic molecules. As the user draws the molecule, an adjacency list is generated to represent the molecule. The "Name My Molecule" button runs an algorithm that generates the name of the molecule. Within the algorithm, many smaller graph theory problems are solved, such as finding the longest path that does not repeat nodes.

Markov Sports Rankings
Previous Next

What: A sports ranking system that takes game scores and strength of schedule into account without introducing the subjectivity associated with coaches polls

Why: To learn jQuery and how to integrate database connections & clock processes in a Heroku application.

How: The website scrapes sports data from Massey Ratings periodically and runs an algorithm to generate team rankings. The algorithm uses a Markov chain to model the strength of teams. To learn more about the algorithm used, feel free to visit the project site, where I explain my modeling in detail.

RunnersDB
Previous Next

What: A web forum in which runners around the world can share their neighborhood's popular running routes.

Why: To learn Node.js, Express.js, and AJAX

How: I used the Google Maps API to allow users to draw and save their custom routes. The API is used to calculate total mileage and an elevation profile for the route. When the user submits a route, it is saved to a database. Users can search the database by location, mileage, elevation change, and user ratings. Users can also save routes to their profiles and comment on others' routes. In order to get comfortable using Node.js, I used as few external modules as I could. For example, I opted to build a username/password authentication myself.
